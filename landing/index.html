Two engines, one place

Claude and Codex, in the same app.

Pick an engine per project or per run, switch anytime — the workflow and history stay exactly the same. You use the subscriptions you already have, no API keys to manage.

Claude

devdy one shared interface

Use your ChatGPT account

All in one app

Manage AI context for every project.

Skills and Rules live in one central library, applied and auto-synced into each project — Devdy spots it whenever a project's copy drifts.

01 · Skill & Rule

Write once, sync everywhere

Write a Skill/Rule once, choose to apply it to Claude, Codex or both, then push it into many projects. Edit the source and it syncs automatically; if a project's copy was hand-edited, Devdy asks whether to overwrite or keep it — no more manual copy-paste.

02 · Analyze issue

Issue → detailed plan

Just enter an issue number. Devdy pulls the content, labels and discussion (skipping bots), then turns it into a detailed plan — following the Skills/Rules you applied to the project.

03 · Review PR

Review pull requests

Enter a PR number and Devdy gathers the changes, comments and linked issues, then reviews them by your conventions.

04 · Free session

Open-ended work sessions

Open a session right inside a project, chat across turns, send more requests while it's running, and @mention files/folders directly.

05 · Stats

Token & cost stats

Track tokens and estimated cost by engine, project, model and by day. Deleting old runs never loses the numbers.

History & concurrency

Always in sync. Runs in parallel.

Devdy gathers every session in one place — even the ones you started from the terminal or VS Code — and lets you run many at once across projects, with none blocking another.

Auto-mirror

Session history from everywhere

Run claude or codex outside the app — in the terminal or VS Code? Devdy automatically detects those sessions and gathers them into your history, no need to open them through the app first. Everything on one unified timeline.

Local-first & privacy

Your data never leaves your machine.

  • No API keys to manage. Engines authenticate with your existing claude / codex CLI subscription logins.
  • Your GitHub token stays safe. It lives only in the OS Keychain, never written to a file or log.
  • Stored entirely on your machine, no cloud. Skills, rules, history and stats all stay local.
  • You're always in control. Devdy asks permission before every action, with several levels of control you choose.
